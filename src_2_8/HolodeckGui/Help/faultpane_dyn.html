Fault Pane Help
Faults are preset error conditions that will help you test your application easily. There are six categories including one to create your own faults to test your application specifically. Faults set and fail an array of tests that simulate common errors.

Tips
You can set a fault at anytime during testing.

You can pause before your application starts to set faults before execution by checking the Pause Application On Start checkbox in the Options Selection page of the New Project Wizard.

Faults will apply an error to an entire category of api calls. To create a more targeted test use the Tests feature, which can target single API calls.

Faults can be applied to all the threads in a process or only to a specific thread.

To easily set a Resource fault right click a resource in the resource pane and select the fault from the context menu..
